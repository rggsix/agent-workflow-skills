# 编写规则

一句话定位：把 skill 写成“短入口 + 必要分文件”，不写成长篇说明书。

## 总原则

- 任何设计都优先“节省 AI 注意力与人类认知成本”。

## 必须

1. `SKILL.md` 只放定位和引用索引，不写长流程。
2. `SKILL.md` 每条引用都要带一句话摘要（1 句话内）。
3. 每个 references 文件开头必须先写一句话定位（这个文件只负责什么）。
4. references 文件“必须有职责才创建”，没有独立职责就不创建。
5. 同一条规则只能出现在一个文件，其他文件只链接不复述。
6. 可执行逻辑放 `scripts/`，模板素材放 `assets/`。

## 禁止

- 为了“看起来完整”而创建多余 references。
- 多个 md 重复同一段话术或同一规则。
- 在 `SKILL.md` 粘贴脚本或大段背景。
- 为了“信息全面”牺牲可读性与可执行性。
- 为了“解释”而解释，不落到可执行约束与步骤。
