# 规则

一句话定位：定义编码后 ToDo 校正的读取与写入边界。

## 必须

1. 仅基于本次执行结果进行新增/完成标记。
2. 复盘实际改动，补充遗漏/隐患/优化点的 ToDo，并标记已完成项。
3. 重点检查与本次需求相关的 ToDo，能闭环的必须标记完成并归档。
4. 按风险分级写入：
   - P0: 不闭环会导致线上风险/阻塞发布
   - P1: 短期必须修，但不立即阻塞
   - P2: 可选优化
5. 写入必须通过 `todo` skill。
6. 若无新增或完成项，直接返回“无更新”。

## 禁止

- 不做新规划。
- 不修改 plan.md。
- 不直接写 todo 文件。
- 只做检查与 ToDo 更新，不修改代码。
