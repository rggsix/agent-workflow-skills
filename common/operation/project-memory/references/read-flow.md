# 读取流程

一句话定位：定义跨会话接续任务时，最小成本恢复工作现场的读取顺序与输出格式。

## 读取顺序

1. 先通过 [Workspace](../../../basic/workspace/SKILL.md) 确认 `project-memory` 对应的官方存储路径。
2. 优先读取 `current.md`，先拿到“现在做到哪”而不是先读历史全量。
3. 读取 `project-profile.md`，补齐项目定位、核心目录和长期稳定事实。
4. 读取 `collaboration-preferences.md`，补齐用户协作偏好与编码习惯。
5. 读取 `decisions.md`，确认最近仍然有效的关键约束与取舍。
6. 最后只读取最近 1 到 3 条 `log/` 记录，用于恢复近期动作、验证和阻塞。

## 输出格式

读取后输出一个简短恢复摘要，至少覆盖以下五项：

1. `Workspace`: 当前仓库、工作目录、正在处理的模块或子目录。
2. `Project`: 这个项目做什么，当前阶段主要目标是什么。
3. `Preferences`: 已确认的协作偏好或编码习惯。
4. `Recent`: 最近完成的改动、验证结果与未闭环风险。
5. `Next`: 建议优先继续的 1 到 3 步。

## 降噪规则

- 若 `current.md` 已足够清晰，不主动回溯更早日志。
- 若状态文件互相冲突，以 `current.md` 的最新日期和 `log/` 的最近记录交叉核对。
- 若缺少关键文件，不补写臆测，直接报告缺口并继续当前任务。
