# 规则

一句话定位：定义 `project-memory` 只负责什么、如何分层，以及绝不能写成什么。

## 规则

1. 写入任何持久化文件前，先通过 [Workspace](../../../basic/workspace/SKILL.md) 获取官方状态域与路径，禁止自行发明目录。
2. 只维护五类记忆：`project-profile.md`、`collaboration-preferences.md`、`current.md`、`decisions.md`、`log/`。
3. `project-profile.md` 只放稳定事实，例如项目目标、核心目录、主要模块、技术栈与运行入口。
4. `collaboration-preferences.md` 只放经明确观察或用户确认过的协作偏好，禁止猜测“用户习惯”。
5. `current.md` 只保留当前阶段最重要的状态：当前目标、正在处理的目录或模块、最近一次验证结果、未完成风险、下一步。
6. `decisions.md` 只沉淀会影响后续工作的决策、约束与明确放弃的路线。
7. `log/` 只做追加式流水账，记录近期做了什么、为什么、改了哪些文件、如何验证、留下了什么风险。
8. 优先恢复现场，不追求“记住一切”；无法帮助下次继续工作的内容不要写。
9. 不替代 [ToDo](../../todo/SKILL.md)；任务优先级、待办生命周期仍归 `todo` 管。
10. 不复制已有文档的大段内容；若事实已存在于代码、配置或其它状态文件，只记录定位方式和必要摘要。
11. 写入时优先具体化：给出目录、文件路径、命令、验证结论与明确日期，避免“已经处理过”“差不多完成”之类空话。
12. 不确定的内容必须显式标记 `Unknown`、`Assumption` 或 `Need verify`，禁止把推测写成事实。

## 边界

- 不把对话逐字转录到记忆文件。
- 不为“看起来完整”而补写低价值背景。
- 不把临时分析草稿放进长期记忆。
