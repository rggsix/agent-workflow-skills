# 规则

一句话定位：只负责识别已完成 ToDo 并归档。

## 必须

1. 使用todo skill读取todo列表。
2. 读取本轮改动范围（`git diff` 或当前git用户最近 commit 列表）。
3. 读取 `coding-workflow` 的 `plan.md`（若存在）。
4. 用关键词/文件路径 名称对照 active ToDo。
5. 产出“完成项清单”，默认自动归档。
6. 仅在匹配不确定时才请求人类确认。
7. 若无匹配项，明确写“无可闭环项”。
8. 归档必须通过 `todo` skill。
9. 若归档项含 `@path:`，归档记录中需保留原路径信息。

## 禁止

- 不新增 ToDo。
- 不修改代码或计划文件。
