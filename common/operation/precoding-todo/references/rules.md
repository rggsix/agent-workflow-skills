# 规则

一句话定位：定义编码前 ToDo 同步的读取与写入边界。

## 必须

1. 只读取 coding-workflow 的 `plan.md`。
2. 复核计划与改动方向，识别“需闭环但当前不做”的风险点/缺口/依赖。
3. 按风险分级写入（precoding 不产出 P2）：
   - P0: 不闭环会导致线上风险/阻塞发布
   - P1: 短期必须修，但不立即阻塞
4. 写入必须通过 `todo` skill。
5. 若 plan.md 不存在或无可提炼项，直接返回“无 ToDo 候选”。

## 禁止

- 不基于 plan 之外的内容生成 ToDo。
- 不把“应当立即完成的工作”当作 ToDo 推迟。
- 不修改 plan.md。
- 不直接写 todo 文件。
