# 规则

一句话定位：定义编码前 ToDo 同步的读取与写入边界。

## 必须

1. 只读取 coding-workflow 的 `plan.md`。
2. 复核计划与改动方向，识别“需闭环但当前不做”的风险点/缺口/依赖。
3. 进入“无 ToDo 候选”结论前，必须先检查本轮是否新增或暴露了用户可触发入口 / 状态入口，以及该入口后的主链路是否完整。
4. 若本轮改动新增或暴露了用户可触发入口 / 状态入口，但后续主链路当前不做、未闭合或仅有占位实现，必须写入 ToDo，不得返回“无 ToDo 候选”。
5. 入口补漏检查至少覆盖以下场景：
   - 新增按钮、菜单、点击项、跳转入口
   - 新增或暴露状态切换入口
   - 入口后存在未实现、空回调、`fatalError`、占位实现、仅切状态无后续动作
   - 其他因本轮改动而从不可达变为可达、且会被用户实际触发的链路未闭合情况
6. 按风险分级写入（precoding 不产出 P2）：
   - P0: 不闭环会导致线上风险/阻塞发布
   - P1: 短期必须修，但不立即阻塞
7. 写入必须通过 `todo` skill。
8. 若 plan.md 不存在或无可提炼项，直接返回“无 ToDo 候选”。

## 禁止

- 不基于 plan 之外的内容生成 ToDo。
- 不把“应当立即完成的工作”当作 ToDo 推迟。
- 不修改 plan.md。
- 不直接写 todo 文件。
