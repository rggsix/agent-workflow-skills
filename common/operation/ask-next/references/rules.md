# 规则

一句话定位：定义 ask-next 的输出与闭环检查边界。

## 必须

1. 读取 `.ai-workspace/todo/active.md`。
2. 按优先级顺序给建议：`P0` -> `P1` -> `P2`。
3. 默认输出 1-3 条建议，不输出全量清单。
4. 若无待办，返回“当前无待办”并给 1 条可选建议。
5. 做一次“深度闭环检查”：结合当前上下文与近期改动，识别可能已完成的 ToDo。
6. 深度检查优先依据 `@path:`、任务关键词与本轮修改范围做匹配，不做主观臆测。
7. 输出中必须包含“闭环检查”小节（即使为空），列出疑似完成项或明确写“无疑似完成项”。
8. 未完成第5-7步，不得给出下一步建议。
9. 对疑似完成项，必须先询问人类确认，再调用 `todo` 归档。

## 禁止

- 未经人类确认不得归档。
- 不创建新任务。
